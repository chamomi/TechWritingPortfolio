5G Network Architecture

What is 5G?

5G is the next generation of network technology. 5G is designed to provide higher connection speed, ultra-low latency and increased bandwidth.

As the technology advancement places higher expectations on the network reliability and response-time, the 4G networks are not able to handle the demand. 5G solutions arise, ready to support a range of services like autonomous vehicles, robotics, VR/AR technology, IoT and many more. Table 1: Comparison of 4G and 5G features

The 5G networks are designed for three general use cases [2]:

  • eMBB (enhanced Mobile Broadband)
    eMBB targets the growing demand for large network bandwidth. The applications include HD video streaming and download, virtual reality and augmented reality technologies that are gaining popularity.
  • URLLC (Ultra-Reliable and Low Latency Communications)
    URLLC aims to ensure the reliable service for critical applications such as autonomous vehicles, manufacturing lines, and remote medical procedures that require low latency and high availability.
  • mMTC (massive Machine Type Communications)
    mMTC scenario’s goal is to support high connection density, as is the case with IoT devices both in the smart home and industrial setting.     

5G Architecture

NR Architecture

5G network architecture consists of a number of gNB or ng-eNB nodes connected to the 5G Core network [3]. Figure 1 presents the NR topology.

The elements within the NG-RAN are connected to each other through the Xn interface. Each node is connected to the core network through the NG interface.

Functional Split

5G introduces the new architecture of the network nodes compared to 4G networks. The gNB consists of a Central Unit (CU) and one or more Distributed Units (DU). gNB-CU is connected to gNB-DUs through the F1 interface [5]. Figure 2 shows the details of the gNB internal architecture.

Figure 2: NG-RAN architecture

The separation of the gNB into CU and several DUs allows moving the gNB-CU to the cloud and virtualization of the network functions performed in the gNB-CU. This solution means lower latency, better load management, real-time performance optimization, independence from the hardware-specific implementations, and adaptation to specific use cases through the configuration of the functional split [3].

The functionality of the gNB-CU and gNB-DU in a specific implementation is defined by the functional split. The functional split can separate the underlying network protocols at 8 points. The options are presented in Figure 3 [3]. Depending on the selected option and virtualized processing in the gNB-CU, the network characteristics will change. So the functional split should be adjusted to the use case requirements. According to the 3GPP recommendations, Option 2 is preferred.

5G Technologies

5G utilizes new technologies to improve the service quality:

  • NFV (Network Functions Virtualization)

NFV brought by gNB functional split makes the 5G network more flexible and adaptable to the user requirements. Centralization and virtualization helps to make network administration and optimization more efficient and less expensive. NFV opens new opportunities to apply software solutions independent of the hardware.

  • New Radio Spectrum

5G network uses new radio frequencies compared to 4G. This increases the available bandwidth.

  • Massive MIMO

Massive MIMO (multiple input, multiple output) approach uses 2D arrays of antennas to amplify the channel efficiency and increase network coverage. Massive MIMO is capable of targeting individual clients both horizontally and vertically, adding more flexibility to the network operation.

With the use of massive MIMO, it is possible to connect more user devices and ensure better connection quality for each of them.

  • Network Slicing

Network slicing technology creates isolated E2E virtual networks on the common physical infrastructure in the network slices. Each slice can have its unique network configurations and can be used as a private network. Network slicing can be a foundation for NaaS model for industries that have specific E2E network requirements.

As network slicing creates separate data pipelines, it is able to guarantee the quality of service for mission-critical services. This will support the development of autonomous devices that require a highly responsive network service.

  • AI-driven Automation

Because 5G network supports NFV, making it more software-oriented, it is possible to apply AI and ML algorithms to automate and optimize the network management. As a result, the network becomes more flexible and efficient.

Summary

5G is the future of networking. Its revolutionary approach to architecture ensures the service of better quality and creates opportunities for new business models. 5G ensures low latency, large bandwidth and unprecedented speed. 5G opens pathways for the wide deployment of applications in new technological fields such as IoT, AR/VR, robotics, autonomous vehicles and many more.

Definitions and Abbreviations

AMF

Access and Mobility Management Function

AR

Augmented Reality

eMBB

enhanced Mobile Broadband

gNB

Next Generation Node B – a node which supports the NR as well as connectivity to 5G Core network

gNB-CU

gNB Central Unit is a logical node hosting a group of higher network protocols according to the functional split. gNB-CU controls the operation of one or more gNB-DUs

gNB-DU

gNB Distributed Unit is a logical node hosting a group of lower network protocols according to the functional split. The operation of gNB-DU is partly controlled by gNB-CU

IoT

Internet of Things

MIMO

Multiple Input, Multiple Output

mMTC

massive Machine-Type Communication

NFV

Network Functions Virtualization

ng-eNB

Next Generation Evolved Node B – a node which supports the E-ULTRA as well as connectivity to 5G Core network

NG-RAN

New Generation Radio Access Network which supports either NR or E-UTRA or both, interfacing with the 5G Core network

NR

New Radio

UPF

User Plane Functions

URLLC

Ultra-Reliable Low-Latency Communication

VR

Virtual Reality
