How to Create a One-to-Many Relationship with Integrity Enforcement in Caspio platform

Introduction

Tables are an integral part of Caspio applications that store the application’s data. The relationships establish the connections between the tables. You can create relationships of three types in the Caspio platform:

  • One-to-one,
  • One-to-many,
  • Undeterminate.

A one-to-many relationship signifies that each record in the parent table relates to one or more records in the child table, but each record in the child table relates to only one record in the parent table.

Note: You can only create relationships between fields that have compatible data types.

For connected tables, it is important to maintain data consistency. Referential integrity ensures that every parent record referenced in the child table exists in the parent table. When integrity is enforced, the system ensures that no child can refer to a parent record that does not exist. Referential integrity is available for one-to-one and one-to-many relationships.

How to Create a One-to-Many Relationship with Integrity Enforcement

To create a one-to-many relationship with integrity enforcement in Caspio:

  1. Open the Tables section of your application.
  2. Click Relationships to open the relationships manager.
  3. Select the tables you want to work with by either:
  • Marking checkboxes in the Tables list.
  • Dragging the tables to the work area.
  1. Open the New Relationship Settings dialog box with one of the following actions:
  • Drag a field from one table in the work area to a field in another table.
    In this case,     Parent Table, Parent Field, Related Table, and Related Field dropdowns are pre-filled automatically.
  • Double-click a field that does not currently have a relationship.
    In this case,     Parent Table and Parent Field dropdowns are pre-filled automatically..
  • Click New Relationship at the top of the screen.
    In this case, no dropdowns are automatically pre-filled.
  1. Configure (if needed) the values for Parent Table, Parent Field, Related Table, and Related Field in the New Relationship Settings dialog box.
  2. Review the new relationship type.

Note: to create a one-to-many relationship, please, make sure that one of the related fields is unique and another one is not.

  1. Select Enforce referential integrity checkbox.
  2. Click Create to create the relationship.
