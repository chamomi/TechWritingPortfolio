Paragon File System 2.0

High performance, low wear, fail-safe, data-critical file system

 for embedded use with flash memory

Paragon File System (Paragon FS, PFS) is a modular file system designed to meet the needs of autonomous and embedded systems. Paragon FS extends flash memory life, provides high performance, and ensures data integrity even in the case of power loss or system failure. Fully deterministic behavior and the ability to work in environments with limited resources allow Paragon FS to operate in RTOSes efficiently. POSIX compliance and a number of adjustable parameters help you to adapt Paragon FS to your use-case and make the most out of the system.

Due to its modular structure, Paragon FS is a perfect choice for pre-OS environments, Automotive, Industrial Autonomous Systems, and IoT use. Paragon FS is already ported to a range of RTOSes, and if your RTOS is not on our list, you can get the system as a set of libraries to integrate them to your RTOS – with Paragon help if required.

Paragon FS Benefits

Fail-Safe Data Integrity

Maintaining data consistency is crucial for a file system operating in an environment where unexpected power failure may occur. With the use of transactions, Paragon FS provides operation reliability and safe recovery of your data at all times. Transactions ensure your data remains atomic, consistent, and isolated. Paragon FS stores all data in the final state, with no intermediates. Atomic file system operations minimize the risk of data corruption.

Copy-on-write mechanism makes Paragon FS even more fault-tolerant by ensuring that data is never overwritten. This approach guarantees that in the case of system failure the data will be consistent and can be easily recovered as initial file contents are safely preserved even if the write transaction was interrupted and not finished correctly.

Even after the power failure or system error, the data can be restored to a valid state. Transactions that were not completed at the time of the device failure will be aborted and your data will be kept intact.

Extending Flash Memory Life

Flash memory has a limit on the number of rewrites per memory block. This causes faster exhaustion of the blocks in case memory operations are performed without special care. Paragon FS is designed to protect your device. To extend your flash memory life, Paragon FS uses dynamic wear-levelling. When performing a write operation, Paragon FS chooses the memory block with the lowest number of erases, spreading the strain evenly.

Striving to further extend the flash memory life, Paragon FS implements copy-on-write approach and metadata pair abstraction. While copy-on-write is primarily used for resource optimization, it also minimizes the number of writes and erases on the flash memory blocks. This way memory wear is slowed down. Similarly, metadata pair pointer minimizes the number of erases in general, protecting memory blocks.

Paragon FS supports both NAND and NOR flash memory. Whatever hardware you use, Paragon FS will provide reliable operation while extending the memory life.

Resources & Performance Optimized

Paragon FS is optimized to work efficiently with limited resources. It can be easily integrated into low-resource OS-like and pre-boot environments. Paragon FS flexibility allows fine-tuning for a wide range of use-case scenarios – from tiny IoT to automotive VCE/ECU hardware. We provide the best performance for the resource requirements of your specific use-case.

Paragon FS is designed to provide optimal performance within the resource limits with the following features:

  • Adjustable bitmap buffer size customizes the file system to your memory constraints. To use minimal resources, bitmap buffer size can be left undefined. If your goal is to improve performance, the bitmap buffer size should be larger.
  • Block cache module provides read/write operations performance improvement by caching blocks of data.
  • Adjustable program/read/erase parameters reduce latency and provide greater flexibility in customizing Paragon FS to specific data type and flash storage.

Paragon FS can be configured to use only 50 KB of RAM. With 64 MB, Paragon FS works faster than FAT.

RTOS Optimized

Paragon FS is designed to provide reliable real-time operation. It is crucial to meet all deadlines in RTOS systems. Execution of recursive code can lead to cycles of undefined length, which is undesirable for real-time systems. To ensure predictable code execution time, Paragon FS does not use any recursive code. You can be sure that all file system operations will be performed within the predicted time.

Paragon FS avoids dynamic memory allocation to run properly and be stable in RTOS environments. Dynamic memory allocation allows changes in memory distribution during the program execution. This makes it unsuitable for RTOSes as real-time systems require that operations are executed with high precision and speed. Despite its other advantages, dynamic memory allocation may slow down the system, leading to unreliable performance in RTOSes. Dynamic memory allocation is also not safe for systems where power loss may occur. In case of system failure, Paragon FS guarantees that data can be easily recovered to a valid state. Static memory allocation, used in Paragon FS, ensures the execution is fast and predictable. Additionally, static memory allocation helps to fulfill AUTOSAR requirements, so PFS can be integrated into demanding automotive systems.

All Paragon FS operations are deterministic, giving you assurance the system will be fully reliable.

POSIX Compliant

Paragon FS provides POSIX-like APIs for convenient development and easy integration with other applications using POSIX-like interfaces. Paragon FS supports well-known Linux APIs as well as ACLs.

Symbolic and hard links ensure easy access to common data. Paragon FS strives to make data manipulation seamless and convenient.

Custom File Attributes

Your unique solution may require the addition of more detailed information about files and directories. Paragon FS allows tailoring the system to your use-case by defining a set of custom file attributes. Attributes extend the system and user data markup, giving you more flexibility.

Fast Search

For many systems, it is important to minimize time of data access. Paragon FS uses tree-based metadata to provide fast data search and metadata versions navigation.

Integration

Paragon FS is available as development libraries and as precompiled drivers for the most popular operating systems. Paragon can help you to port PFS to your environment if needed.

Paragon FS supports FUSE integration to Linux. You can test it as a part of our Starter Kit and get more details in the Developer’s Guide coming with the full version.

Paragon FS can be integrated with other storage media like HDD and SD memories with the implementation of the appropriate interface.
